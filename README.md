# 낱말바둑 — 모두가 함께하는 십자낱말

모두가 함께 하나의 20x20 보드를 채워가는 십자낱말 + 끝말잇기 웹게임.
바둑처럼 글자로 빈 칸을 완전히 둘러싸면 그 땅을 차지한다.
보드는 Firebase Realtime Database에 영구 저장되어 지워지지 않는다.

## 구조 (Firebase)

- **Hosting**: `public/` (게임 페이지 + 사전 `words.txt`)
- **Realtime Database**: 보드 상태(`/board`), 접속자(`/presence`)
  - 실시간 리스너로 다른 유저의 단어와 실시간 입력 상황이 즉시 반영됨
  - 배치는 RTDB 트랜잭션으로 처리해 동시 입력 경합에도 안전
- **Cloud Functions**: `functions/` — IP 기반 익명 플레이어 ID 발급 (`/api/identify`)
  - 실제 IP는 함수 안에서만 잠깐 쓰이고 salted HMAC 해시로 변환한 짧은 익명 ID(`P-xxxxxxxxxx`)만 응답
  - IP 원문은 클라이언트·DB 어디에도 저장되지 않음
- 그 외 서버 코드 없음 — 검증(사전·두음법칙·중복·포획)은 클라이언트 + 트랜잭션 재검증

## 배포 (GitHub Actions 자동)

`main`에 푸시하면 `.github/workflows/deploy.yml`이 자동으로 Firebase에 배포한다.

필요한 설정 (1회):
1. Firebase 콘솔에서 **Realtime Database 생성** (Build → Realtime Database → 데이터베이스 만들기)
2. **Blaze(종량제) 플랜으로 전환** — Cloud Functions 배포에 필수
   (Firebase 콘솔 → 프로젝트 설정 → 사용량 및 결제 → 플랜 업그레이드)
   업그레이드 전까지는 `/api/identify` 호출이 실패하고, 클라이언트가 자동으로
   기존 방식(브라우저별 임의 ID)으로 폴백해서 게임 자체는 계속 정상 동작한다.
3. GitHub 레포 **Settings → Secrets and variables → Actions → New repository secret**
   - Name: `FIREBASE_SERVICE_ACCOUNT`
   - Value: Firebase 콘솔 → 프로젝트 설정 → 서비스 계정 → 새 비공개 키(JSON) 내용
4. (권장) IP 해시용 SALT 별도 설정 — 기본값은 소스에 공개돼 있어 안전하지 않음
   GitHub 레포 **Settings → Secrets and variables → Actions → New repository secret**
   - Name: `IDENTIFY_SALT`
   - Value: 충분히 긴 랜덤 문자열 (예: `openssl rand -hex 32` 로 생성)

   배포 워크플로가 이 시크릿으로 `functions/.env`를 자동 생성한다.
   (`firebase functions:config:set`은 2025년 말 셧다운된 구식 API라 쓰지 않음)

배포 시 `scripts/gen-firebase-config.js`가 웹앱 등록과 `public/firebase-config.js` 생성을 자동 처리한다.

## 로컬 실행

배포된 DB를 그대로 쓰면서 페이지만 로컬로 띄우기:

```bash
# public/firebase-config.js가 필요 (배포 후 호스팅에서 받아오거나 CI 아티팩트 사용)
npx serve public   # 또는 python3 -m http.server -d public 8000
```

`/api/identify`는 로컬 서버에는 없으므로, 로컬 실행 시에는 자동으로 브라우저별 임의 ID로 폴백한다.

## 게임 규칙

- **사전 검증**: 명사 203,513개(`public/words.txt`)에 있는 단어만 가능
  (표준국어대사전 + hunspell-dict-ko 병합, 옛말·북한어·방언 제외
  + 한국 행정구역/세계 주요 국가·도시명/유명 브랜드명/유명 인물명 약 400여 개 추가)
- **두음법칙**: 기존 글자에서 시작할 때 첫 글자의 두음 변환형 인정 (ㄹ→ㄴ/ㅇ, ㄴ→ㅇ)
  - 원형 입력: '력' 칸에서 '력사' 입력 → '역사'로 인정
  - 변환형 입력: '립' 칸을 클릭해 '입구'라고 입력해도 인정 (그리드에는 '립구'로 표기)
- **방향 제한 없음**: 가로·세로 어느 칸에서든 같은 방향으로 계속 이어 쓸 수 있음
- **단어 중복 금지**: 보드 전체에서 같은 단어(두음법칙 인정형 기준)는 한 번만 쓸 수 있음
- **병합/분리 판독**: 같은 줄에 붙은 기존 글자는 합쳐 읽을 수도, 분리해 읽을 수도 있음.
  [앞글자+입력+뒷글자, 앞글자+입력, 입력+뒷글자, 입력] 순(긴 판독부터)으로 검사해
  규칙과 사전을 만족하는 첫 판독을 채택
- 교차 글자는 일치해야 하고, 최소 한 글자는 기존 단어와 이어져야 함 (첫 단어 예외)
- 완성 단어 기준 2~15글자

## 땅따먹기 (영역 포획)

- 글자로 완전히 둘러싸인 빈 칸(상하좌우 4방향 기준)은 그 순간 자동으로 **포획**됨
- 포획한 사람은 그 영역을 마지막으로 둘러싼(완성시킨) 유저
- 포획된 칸은 글자가 없는 영역으로 영구 비활성화 — 이후 누구도 그 칸을 지나는 단어를 놓을 수 없음
- 우측 상단 리더보드에 유저별 포획 칸 수 기준 순위 표시 (자신은 자기 색으로 강조)

## 플레이어 식별 · 색상

- 플레이어는 IP 주소 기반 익명 ID(`P-xxxxxxxxxx`)로 구분됨 (`functions/index.js` 참고)
- 같은 공인 IP를 쓰는 사람(같은 와이파이, 같은 통신사 NAT 등)은 같은 ID로 묶이고,
  네트워크를 바꾸면(와이파이↔모바일데이터, VPN 등) 다른 ID가 됨
- 각 플레이어는 ID로부터 계산된 고유 색을 가지며, 이 색이 아래 전부에 일관되게 쓰임:
  선택 칸 하이라이트, 입력 중인 글자 미리보기, 포획한 영역, 리더보드
- Cloud Functions 미배포/실패 시엔 브라우저별 임의 ID로 자동 폴백

## 실시간 참여 상황

- 누구나 게임 중간에 들어와도 다른 유저가 지금 어느 칸을 선택했고 무엇을 입력 중인지
  실시간으로 각자 색깔의 옅은 글자로 보임 (`/presence`에 브로드캐스트)
- 15초 이상 갱신이 없는 상태는 자동으로 화면에서 제거됨 (비정상 종료 등 대비)

## 조작

- 칸 클릭 → 바로 타이핑 (보이는 입력창 없이 글자가 캔버스 칸에 바로 그려짐)
- Tab: 가로/세로 모드 전환 (칸 위 작은 배지로 현재 모드 표시)
- 방향키: 선택 칸을 인접 칸으로 이동
- 드래그로 이동, 휠로 확대/축소, Enter로 제출, Escape로 취소
- 미리보기: 내 색 = 새 글자, 초록 = 함께 읽히는 기존 글자, 빨강 = 충돌/에러

## 알려진 한계 (프로토타입)

- 검증이 클라이언트에서 수행되므로 악의적 유저가 DB에 직접 쓰는 것까지 막지는 못함
  (강화하려면 Cloud Functions 검증 + 규칙 잠금으로 이전)
- 보드가 커지면 트랜잭션이 전체 보드를 전송하므로 규모 확장 시 셀 단위 구조로 개편 필요
- 보드는 20x20 고정 크기(좌표 -10~9)이며, 범위 밖은 항상 벽으로 취급되어 글자를 놓거나 이어붙일 수 없음
- 포획 탐지는 안전장치로 한 번에 가로/세로 폭 17칸까지만 탐색(그보다 넓게 뻗어나가는 영역은 포획하지 않음)
- IP 기반 식별은 같은 네트워크를 쓰는 여러 사람을 한 플레이어로 묶고, 반대로 한 사람이
  네트워크를 바꾸면 다른 플레이어로 갈라짐 — 완벽한 개인 식별은 아님
